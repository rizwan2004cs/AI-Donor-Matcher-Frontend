# Feature 8.3 — PWA Manifest

> **Branch:** `feat/pwa-manifest`
> **File:** `public/manifest.json`, `index.html` link

---

## Overview

A Web App Manifest that allows the app to be installed on mobile home screens. Defines the app name, icons, theme colour, and start URL for the PWA.

---

## Files to Create / Modify

| File | Action |
|------|--------|
| `public/manifest.json` | Create |
| `index.html` | Add `<link rel="manifest">` |
| `public/icon-192.png` | Add (192×192 app icon) |
| `public/icon-512.png` | Add (512×512 app icon) |

---

## Manifest File

```json
{
  "short_name": "DonorMatch",
  "name": "AI Donation Matcher",
  "icons": [
    {
      "src": "/icon-192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "/icon-512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ],
  "start_url": "/",
  "display": "standalone",
  "theme_color": "#4F46E5",
  "background_color": "#ffffff"
}
```

---

## index.html Changes

Add inside `<head>`:

```html
<link rel="manifest" href="/manifest.json" />
<meta name="theme-color" content="#4F46E5" />
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-status-bar-style" content="default" />
<link rel="apple-touch-icon" href="/icon-192.png" />
```

---

## Icon Requirements

| Size | Usage |
|------|-------|
| 192×192 | Android home-screen icon, splash screen |
| 512×512 | Android splash screen, PWA install prompt |

> Generate icons from the app logo. Use a tool like [realfavicongenerator.net](https://realfavicongenerator.net/) or export from Figma.

---

## Behaviour Notes

- `display: "standalone"` hides the browser chrome when launched from home screen.
- `theme_color` tints the status bar on Android (indigo-600 = `#4F46E5`).
- `start_url: "/"` ensures the app always opens on the Discovery Map.
- The manifest must be served from the same origin. Vite copies everything in `/public` to the build root.

| Criterion | How We Meet It |
|-----------|---------------|
| Has a `<link rel="manifest">` | ✓ in index.html |
| `name` and `short_name` set | ✓ manifest.json |
| Icons ≥ 192px and ≥ 512px | ✓ two icons |
| `start_url` set | ✓ `/` |
| `display` is `standalone` or `fullscreen` | ✓ `standalone` |
| `theme_color` set | ✓ `#4F46E5` |
| `background_color` set | ✓ `#ffffff` |
| Served over HTTPS | ✓ Vercel provides HTTPS |

## Acceptance Criteria

- [ ] `manifest.json` is accessible at `/manifest.json` in the browser
- [ ] Chrome DevTools → Application → Manifest shows correct values
- [ ] Icons load without 404
- [ ] "Add to Home Screen" prompt appears on Android (after service worker is registered)
- [ ] Launched from home screen → standalone mode (no address bar)
- [ ] Status bar colour matches theme colour

3. On Android Chrome → verify install banner or "Add to Home Screen" option
4. Launch from home screen → standalone display
5. Check `/icon-192.png` and `/icon-512.png` load in browser
