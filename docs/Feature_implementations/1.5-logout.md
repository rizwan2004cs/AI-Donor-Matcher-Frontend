## AuthContext `logout()` Implementation

Already present in boilerplate:

```js
const logout = () => {
  localStorage.removeItem("token");
  localStorage.removeItem("user");
  setUser(null);
};
```

## Navbar Integration

```jsx
import { useAuth } from "../auth/AuthContext";
import { useNavigate, Link } from "react-router-dom";

const Navbar = () => {
  const { user, logout } = useAuth();
  const navigate = useNavigate();

  const handleLogout = () => {
    logout();
    navigate("/login");
  };

  return (
    <nav className="bg-white shadow px-6 py-3 flex items-center justify-between">
      <span className="text-xl font-bold text-indigo-600">
        AI Donation Matcher
      </span>

      {user && (
        <div className="flex items-center gap-4">
          {user.role === "DONOR" && (
            <>
              <Link to="/">Discover</Link>
              <Link to="/dashboard/donor">My Pledges</Link>
            </>
          )}
          {user.role === "NGO" && (
            <Link to="/dashboard/ngo">Dashboard</Link>
          )}
          {user.role === "ADMIN" && (
            <Link to="/dashboard/admin">Admin</Link>
          )}

          <span className="text-sm text-gray-500">{user.fullName}</span>
          <button
            onClick={handleLogout}
            className="bg-red-500 text-white px-4 py-1.5 rounded hover:bg-red-600"
          >
            Logout
          </button>
        </div>
      )}

      {!user && (
        <div className="flex items-center gap-4">
          <Link to="/login">Login</Link>
          <Link to="/register">Register</Link>
        </div>
      )}
    </nav>
  );
};

export default Navbar;
```

---

## Behaviour

1. User clicks **Logout** button in Navbar
2. `logout()` clears `token` and `user` from localStorage
3. `setUser(null)` triggers re-render — Navbar hides authenticated links
4. `navigate("/login")` redirects to login page
5. Any ProtectedRoute now redirects to `/login` because `user` is null

---

## Security Notes

- The backend does **not** have a logout endpoint (JWT is stateless)
- Token is invalidated client-side by removing from localStorage
- Axios interceptor will no longer attach a token → subsequent API calls return 401
