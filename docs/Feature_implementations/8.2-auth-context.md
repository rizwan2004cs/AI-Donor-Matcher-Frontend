# Feature 8.2 — Auth Context

> **Branch:** `feat/auth-context`

## Overview

A React Context that holds the current user object and JWT token. Provides `login()` and `logout()` helpers and a `useAuth()` hook so any component in the tree can read auth state and trigger auth actions.

## Component Specification

### AuthContext.jsx

```jsx
import { createContext, useContext, useState } from "react";

const AuthContext = createContext(null);

export function AuthProvider({ children }) {
  const [user, setUser] = useState(() => {
    const stored = localStorage.getItem("user");
    return stored ? JSON.parse(stored) : null;
  });

  const login = (userData, token) => {
    localStorage.setItem("user", JSON.stringify(userData));
    localStorage.setItem("token", token);
    setUser(userData);
  };

  const logout = () => {
    localStorage.removeItem("user");
    localStorage.removeItem("token");
    setUser(null);
  };

  return (
    <AuthContext.Provider value={{ user, login, logout }}>
      {children}
    </AuthContext.Provider>
  );
}

export function useAuth() {
  return useContext(AuthContext);
}
```

### main.jsx Integration

```jsx
import React from "react";
import ReactDOM from "react-dom/client";
import { BrowserRouter } from "react-router-dom";
import { AuthProvider } from "./auth/AuthContext";
import App from "./App";
import "./index.css";

ReactDOM.createRoot(document.getElementById("root")).render(
  <React.StrictMode>
    <BrowserRouter>
      <AuthProvider>
        <App />
      </AuthProvider>
    </BrowserRouter>
  </React.StrictMode>
);
```

## User Object Shape

The `userData` object stored after login matches the backend response:

```json
{
  "id": 1,
  "fullName": "Rahul Sharma",
  "email": "rahul@example.com",
  "role": "DONOR"
}
```

Role values: `"DONOR"`, `"NGO"`, `"ADMIN"`.

## Usage Examples

```jsx
import { useAuth } from "../auth/AuthContext";

function SomeComponent() {
  const { user, login, logout } = useAuth();

  // Check if logged in
  if (!user) return <p>Please log in</p>;

  // Read role
  if (user.role === "ADMIN") { /* ... */ }

  // After API login call
  const handleLogin = async (formData) => {
    const res = await API.post("/api/auth/login", formData);
    login(res.data.user, res.data.token);
  };

  // Logout
  const handleLogout = () => {
    logout();
    navigate("/login");
  };
}
```

## State Persistence

```
Login flow:
  API returns { user, token }
        │
        ▼
  login(user, token)
        │
        ├──▶ localStorage.setItem("user", JSON.stringify(user))
        ├──▶ localStorage.setItem("token", token)
        └──▶ setUser(user)   →   re-render

Page refresh:
  useState(() => JSON.parse(localStorage.getItem("user")))
        │
        ▼
  user restored from localStorage (no API call)

Logout:
  logout()
        │
        ├──▶ localStorage.removeItem("user")
        ├──▶ localStorage.removeItem("token")
        └──▶ setUser(null)   →   re-render → ProtectedRoute redirects
```

---

## Behaviour Notes

- State is initialised from `localStorage` so a page refresh doesn't log the user out.
- The JWT token itself is **not** stored in React state — it's read from `localStorage` by the Axios interceptor at request time.
- `login()` and `logout()` are stable references (defined inside the provider) so they don't cause unnecessary re-renders.
- Context value `{ user, login, logout }` is a new object each render. If performance becomes an issue, memoize with `useMemo`.
