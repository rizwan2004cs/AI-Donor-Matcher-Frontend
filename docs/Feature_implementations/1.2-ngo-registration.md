# Feature 1.2 — NGO Registration

> **Branch:** `feat/ngo-registration`
> **File:** `src/pages/Register.jsx` (shared with Donor registration)
> **Route:** `/register`
> **Depends on:** Axios (`src/api/axios.js`)

---

## Overview

When the user selects the **NGO** role on the registration page, an additional file upload field appears for uploading credentials (registration certificates). The form submits as `multipart/form-data` to include the file.

After successful registration, the NGO profile is set to `PENDING` status. The NGO must wait for admin verification before they can complete their profile and appear on the donor map.

## API Contract

```
POST /api/auth/register
Content-Type: multipart/form-data

Form Fields:
  fullName:   "Hope Foundation"
  email:      "info@hopefdn.org"
  password:   "SecurePass123"
  location:   "Chennai, Tamil Nadu"
  role:       "NGO"
  documents:  <File: registration_certificate.pdf>

Response (201):
{ "message": "Registration successful. Your application is under review." }

Error (400):
{ "message": "Document upload is required for NGO registration" }

Error (409):
{ "message": "Email already registered" }
```

## Component Specification

### Conditional File Upload Field

Only render when `role === "NGO"`:

```jsx
{role === "NGO" && (
  <div>
    <label className="block text-sm font-medium text-gray-700 mb-1">
      Upload Credentials (PDF or image)
    </label>
    <input
      type="file"
      accept=".pdf,image/*"
      onChange={(e) => setFile(e.target.files?.[0] || null)}
      className="w-full text-sm text-gray-500 file:mr-4 file:py-2 file:px-4
                 file:rounded-lg file:border-0 file:text-sm file:font-medium
                 file:bg-blue-50 file:text-[#2E75B6] hover:file:bg-blue-100"
    />
    {file && (
      <p className="text-xs text-gray-400 mt-1">📎 {file.name}</p>
    )}
  </div>
)}
```

### FormData Construction for NGO

```js
const formData = new FormData();
formData.append("fullName", fullName);
formData.append("email", email);
formData.append("password", password);
formData.append("location", location);
formData.append("role", "NGO");
formData.append("documents", file);

await api.post("/api/auth/register", formData, {
  headers: { "Content-Type": "multipart/form-data" },
});
```

### Client-Side File Validation

Before appending to FormData, validate:

```js
const MAX_SIZE = 5 * 1024 * 1024; // 5 MB
const ALLOWED_TYPES = ["application/pdf", "image/jpeg", "image/png"];

if (role === "NGO") {
  if (!file) {
    setError("Please upload your registration certificate");
    return;
  }
  if (!ALLOWED_TYPES.includes(file.type)) {
    setError("Only PDF, JPG, and PNG files are accepted");
    return;
  }
  if (file.size > MAX_SIZE) {
    setError("File size must be under 5 MB");
    return;
  }
}
```

## Post-Registration Flow

1. NGO submits registration → backend creates account with `status: PENDING`
2. Admin receives email notification about new NGO application
3. NGO is redirected to the `/verify-email` screen, where they enter the OTP code sent to their email
4. After OTP verification, NGO can log in but sees a "Pending Verification" message
5. Once admin approves → NGO is redirected to `/ngo/complete-profile`
