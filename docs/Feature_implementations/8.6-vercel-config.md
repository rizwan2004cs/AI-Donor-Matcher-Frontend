# Feature 8.6 — Vercel Deployment Config

> **Branch:** `feat/vercel-config`
> **File:** `vercel.json`

---

## Overview

A Vercel configuration file that rewrites all non-file requests to `index.html`, enabling client-side routing with react-router-dom. Also covers environment variable setup in the Vercel dashboard.

---

## Files to Create / Modify

| File | Action |
|------|--------|
| `vercel.json` | Create |

---

## Configuration File

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

This single rewrite rule catches every URL path and serves `index.html`, letting react-router handle routing on the client.

---

## Environment Variables (Vercel Dashboard)

Set these in **Project Settings → Environment Variables**:

| Variable | Example Value | Scope |
|----------|---------------|-------|
| `VITE_API_BASE_URL` | `https://ai-donor-matcher-api.onrender.com` | Production |
| `VITE_OSRM_URL` | `https://router.project-osrm.org` | Production |

> Vite inlines `VITE_*` variables at **build time**, so changes require a re-deploy.

## Deployment Flow

```
git push origin main
        │
        ▼
  Vercel detects push (GitHub integration)
        │
        ▼
  npm install → npm run build → output /dist
        │
        ▼
  Static files served from Vercel CDN
        │
        ▼
  All routes → /index.html (SPA rewrite)
```

---

## Behaviour Notes

- Without the rewrite, refreshing on `/dashboard/donor` would return a 404 from Vercel.
- The rewrite only applies to paths that don't match a real file (Vercel checks files first).
- Environment variables prefixed with `VITE_` are baked into the JS bundle at build time — they are **not** server-side secrets.
- For preview deployments (PRs), set variables for the "Preview" scope too.
