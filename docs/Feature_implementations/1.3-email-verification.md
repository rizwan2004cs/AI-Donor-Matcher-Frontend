## API Contract

```
POST /api/auth/resend-verification
Content-Type: application/json

Request Body:
{ "email": "rahul@gmail.com" }

Response (200):
{ "message": "Verification email resent" }

Error (404):
{ "message": "Email not found" }

Error (400):
{ "message": "Email already verified" }
```

## Component Specification

### State

```js
const [resending, setResending] = useState(false);
const [resent, setResent] = useState(false);
```

### Getting the Email

The email can come from:
1. `location.state?.email` — passed from the Register page via `navigate("/verify-email", { state: { email } })`
2. As a fallback, prompt or show a generic message

### Resend Handler

```js
const handleResend = async () => {
  setResending(true);
  try {
    await api.post("/api/auth/resend-verification", { email });
    setResent(true);
  } catch (err) {
    alert(err.response?.data?.message || "Failed to resend");
  } finally {
    setResending(false);
  }
};
```
