## Overview

An email verification screen shown after registration. Instead of clicking a verification link, the user receives a **one-time passcode (OTP)** at their email address and types it into this screen. Includes a **Resend code** action.

## API Contract

There are two key endpoints used by this screen:

### 1. Send / Resend OTP

```
POST /api/auth/send-otp
Content-Type: application/json

Request Body:
{ "email": "rahul@gmail.com" }

Response (200):
{ "message": "Verification code sent" }

Error (404):
{ "message": "Email not found" }

Error (400):
{ "message": "Email already verified" }
```

This endpoint can be called immediately after registration (server-side) and again from the **Resend code** button on this screen.

### 2. Verify OTP

```
POST /api/auth/verify-otp
Content-Type: application/json

Request Body:
{ "email": "rahul@gmail.com", "otp": "123456" }

Response (200):
{ "message": "Email verified" }

Error (400):
{ "message": "Invalid or expired code" }
```

## Component Specification

### State

```js
const [email, setEmail] = useState("");
const [otp, setOtp] = useState("");
const [resending, setResending] = useState(false);
const [verifying, setVerifying] = useState(false);
const [verified, setVerified] = useState(false);
const [message, setMessage] = useState(null);
const [error, setError] = useState(null);
```

### Getting the Email

The email can come from, in priority order:

1. Query string: `?email=rahul@gmail.com`
2. `location.state?.email` — passed from the Register page via `navigate("/verify-email", { state: { email } })`
3. `localStorage.getItem("pendingVerificationEmail")`

If no email can be resolved, show a friendly message asking the user to register again.

### Verify Handler

```js
const handleVerify = async (e) => {
  e.preventDefault();
  if (!email || otp.length < 4) {
    setError("Please enter the verification code sent to your email.");
    return;
  }

  setVerifying(true);
  setError(null);
  setMessage(null);

  try {
    await api.post("/api/auth/verify-otp", { email, otp });
    setVerified(true);
    setMessage("Email verified successfully! You can now log in.");
    localStorage.removeItem("pendingVerificationEmail");
  } catch (err) {
    setError(
      err.response?.data?.message ||
        "Invalid or expired code. Please try again or resend."
    );
  } finally {
    setVerifying(false);
  }
};
```

### Resend Handler

```js
const handleResend = async () => {
  if (!email) {
    setError("Could not resend — please register again.");
    return;
  }
  setResending(true);
  setError(null);
  setMessage(null);
  try {
    await api.post("/api/auth/send-otp", { email });
    setMessage("Verification code sent. Check your inbox.");
  } catch (err) {
    setError(
      err.response?.data?.message ||
        "Could not resend code. Please try again later."
    );
  } finally {
    setResending(false);
  }
};
```

## Behaviour Notes

- Check `user.emailVerified === false` from the user object stored in AuthContext
