## Overview

NGOs can mark a locked need (one with active pledges) as fulfilled. This triggers a backend cascade: trust score recalculation, donor thank-you emails, and the need moves to fulfilled history.

## API Contract

```
PATCH /api/needs/{id}/fulfill

Response (200):
{
  "message": "Need marked as fulfilled",
  "newTrustScore": 82,
  "newTrustLabel": "Trusted"
}
```

---

## Component Specification

### Handler

```js
const handleFulfill = async (needId) => {
  if (!confirm("Mark this need as fulfilled? This action cannot be undone.")) return;

  try {
    const res = await api.patch(`/api/needs/${needId}/fulfill`);

    // Remove from active needs list
    setNeeds((prev) => prev.filter((n) => n.id !== needId));

    // Update trust score in header
    setTrustScore(res.data.newTrustScore);
    setTrustLabel(res.data.newTrustLabel);

    // Increment fulfilled count
    setFulfilledCount((prev) => prev + 1);
  } catch (err) {
    alert(err.response?.data?.message || "Failed to mark as fulfilled");
  }
};
```

### Button (within need card, see 5.2)

```jsx
{hasPledges && (
  <button
    onClick={() => handleFulfill(need.id)}
    className="bg-green-600 text-white px-3 py-1.5 rounded text-sm hover:bg-green-700"
  >
    ✅ Mark as Fulfilled
  </button>
)}
```

---

## Backend Side Effects (for reference)

When `PATCH /api/needs/{id}/fulfill` is called, the backend:

1. Marks the need as `FULFILLED`
2. Marks all associated pledges as `FULFILLED`
3. Recalculates the NGO's trust score
4. Sends thank-you emails to all donors who pledged
5. Records in fulfilled history

---

## Behaviour Notes

- Button only appears on needs with `quantityPledged > 0`
- Confirmation dialog required before action
- After fulfillment, the need disappears from "Active Needs" and appears in history
- Trust score and tier update in the dashboard header immediately
- Action is irreversible
