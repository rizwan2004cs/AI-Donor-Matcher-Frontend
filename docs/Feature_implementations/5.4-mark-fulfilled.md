# 5.4 Receive Pledge

Last updated: 2026-03-31

## Old Assumption Removed

The NGO frontend should no longer treat receipt of one donor delivery as full need fulfillment.

## Current Contract

- `PATCH /api/ngo/my/pledges/{pledgeId}/receive`

## Behavior

- only `ACTIVE` incoming pledges are receivable
- receiving a pledge marks that pledge fulfilled
- need totals are recalculated
- the need remains open until total received quantity reaches the full requirement
