# Feature 8.5 — Offline Connectivity Check

> **Branch:** `feat/offline-check`
> **Files:** `src/hooks/useOnlineStatus.js`, integration in mutating components

---

## Overview

A custom hook that tracks `navigator.onLine` in real time. Components performing mutating actions (pledge, report, fulfill) check this flag and show a toast/banner instead of making a request that will fail silently.

## Hook Specification

### useOnlineStatus.js

```js
import { useState, useEffect } from "react";

export default function useOnlineStatus() {
  const [online, setOnline] = useState(navigator.onLine);

  useEffect(() => {
    const goOnline = () => setOnline(true);
    const goOffline = () => setOnline(false);

    window.addEventListener("online", goOnline);
    window.addEventListener("offline", goOffline);

    return () => {
      window.removeEventListener("online", goOnline);
      window.removeEventListener("offline", goOffline);
    };
  }, []);

  return online;
}
```

## Usage — Offline Banner

Display a persistent banner at the top of the page when offline:

```jsx
import useOnlineStatus from "../hooks/useOnlineStatus";

function App() {
  const online = useOnlineStatus();

  return (
    <>
      {!online && (
        <div className="bg-yellow-400 text-yellow-900 text-center text-sm py-1 font-medium">
          You are offline. Some features may be unavailable.
        </div>
      )}
      <Navbar />
      {/* ... routes */}
    </>
  );
}
```

## Usage — Guard Mutating Actions

```jsx
import useOnlineStatus from "../hooks/useOnlineStatus";

function PledgeScreen() {
  const online = useOnlineStatus();

  const handlePledge = async () => {
    if (!online) {
      alert("You are offline. Please reconnect to submit your pledge.");
      return;
    }
    // proceed with API call
    await API.post(`/api/pledges`, payload);
  };

  return (
    <button
      onClick={handlePledge}
      disabled={!online}
      className={`px-4 py-2 rounded text-white ${
        online ? "bg-indigo-600 hover:bg-indigo-700" : "bg-gray-400 cursor-not-allowed"
      }`}
    >
      Pledge Now
    </button>
  );
}
```

## Behaviour Notes

- `navigator.onLine` is a browser API; it may report `true` even if the actual internet is down (e.g., connected to WiFi with no internet). It's a best-effort check.
- The `online` / `offline` events fire when the browser detects connectivity changes.
- Read-only pages (Discovery Map, NGO Profile) still work offline if the service worker cached the API responses.
- Mutating endpoints should always be guarded because they **will** fail without connectivity.
