# Feature 7.5 — Protected Route

> **Branch:** `feat/protected-route`
> **File:** `src/components/ProtectedRoute.jsx`
> **Depends on:** AuthContext

---

## Overview

A route-guard higher-order component that wraps protected page routes. Checks if the user is authenticated and optionally if they have the required role. Redirects unauthenticated users to `/login` and unauthorised users to `/`.

## Component Specification

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | The page component to render |
| `role` | `string` | `null` | Required role, e.g. `"DONOR"`, `"NGO"`, `"ADMIN"`. If `null`, any authenticated user passes. |

### Full Component

```jsx
import { Navigate } from "react-router-dom";
import { useAuth } from "../auth/AuthContext";

export default function ProtectedRoute({ children, role }) {
  const { user } = useAuth();

  // Not logged in → login page
  if (!user) return <Navigate to="/login" replace />;

  // Logged in but wrong role → home page
  if (role && user.role !== role) return <Navigate to="/" replace />;

  // Authorised
  return children;
}
```

## Usage in App.jsx

```jsx
import ProtectedRoute from "./components/ProtectedRoute";
import DonorDashboard from "./pages/DonorDashboard";
import NgoDashboard from "./pages/NgoDashboard";
import AdminDashboard from "./pages/AdminDashboard";

<Route
  path="/dashboard/donor"
  element={
    <ProtectedRoute role="DONOR">
      <DonorDashboard />
    </ProtectedRoute>
  }
/>

<Route
  path="/dashboard/ngo"
  element={
    <ProtectedRoute role="NGO">
      <NgoDashboard />
    </ProtectedRoute>
  }
/>

<Route
  path="/dashboard/admin"
  element={
    <ProtectedRoute role="ADMIN">
      <AdminDashboard />
    </ProtectedRoute>
  }
/>
```

## Behaviour Notes

- Uses `<Navigate replace />` so the protected URL is **not** pushed onto the history stack; pressing back won't loop.
- If no `role` prop is given, the guard only checks authentication (any logged-in user can access).
- Role values are uppercase strings matching the backend JWT claim: `DONOR`, `NGO`, `ADMIN`.
- On token expiry (detected by Axios 401 interceptor clearing auth), the next render cycle will redirect automatically.

## Acceptance Criteria

- [ ] Unauthenticated user → redirect to `/login`
- [ ] Authenticated user with wrong role → redirect to `/`
- [ ] Authenticated user with correct role → renders children
- [ ] No `role` prop → any authenticated user passes
- [ ] Uses `replace` navigation (no history loop)
